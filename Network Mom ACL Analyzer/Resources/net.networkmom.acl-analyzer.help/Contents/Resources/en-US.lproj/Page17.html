Feature Overview

Supported OS Variants

As of version 2.0 (18 October 2019) Network Mom ACL Analyzer supports analyzing the following variants of Cisco IPv4 and IPv6 ACLs:

  • Cisco IOS/IOS-XE (including network object-groups but not service object-groups)

  • Cisco IOS-XR (including object-groups)

  • Cisco NX-OS (including object-groups)

  • Cisco ASA (including network object-groups but not service object-groups). Note that ASA support combined IPv4/IPv6 functionality in one ACL (ASA versions 9.0 and later).

Every time you perform an analysis, make sure to set the  device type . If every line of your ACL is reported as an error, this is the most likely cause.

ACL Analysis -> Permit-Deny Analysis -> Validate ACLs

This “Validate ACLs” button performs the following functions:

  • Tests the ACL for basic syntax

  • Reports misaligned subnetting (such as 10.1.1.1/24) as a warning.

  • Reports discontiguous or misaligned “do-not-care-bits” (such as “10.1.1.0 0.0.0.253”) as an error.

  • Warns if multiple ACL names are detected in the input.

  • Warns if sequence numbers are out-of-order

ACL Analysis -> Permit-Deny Analysis -> Analyze Socket

In addition to revalidating the ACL, the Analyze Socket command can test to see if a specific TCP or UDP socket is permitted or denied through the acl.

ACL Analysis -> Duplicate ACL Finder -> Search For Duplicates

In addition to revalidating the ACL, the “search for duplicates” command finds ACL lines which will never be matched because a prior line permits everything the found line matches.

This may be the result of an unnecessary second line. But it also might mean the first line is too permissive. If the first line is “permit ip any any”, every subsequent line will be reported as a duplicate. But removing all those duplicates and leaving “permit ip any any” in place may be a security risk. This tool helps to find these situations, but the decision what to do about it is up to your network, information security, and management teams.

The time required to search for duplicates increases exponentially O(N^2) as the number of lines increases. A 2000-line ACL can be searched for duplicates in a few seconds. But a 10,000-line ACL will take 25 times as long (several minutes).

ACL Utilities -> Random ACL Generation

To make it easy to try out the permit-deny analysis and duplicate ACL detection, we’ve included utilities to create 2,000-line “random” access-lists for each of the four supported platforms.

These are, of course, only intended for demonstrating this tool. Do not use random access-lists in your production environments!
