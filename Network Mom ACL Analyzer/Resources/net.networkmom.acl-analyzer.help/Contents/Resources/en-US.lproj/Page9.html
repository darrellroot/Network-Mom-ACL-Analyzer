Terminology

Socket : The specification for a network communication. An IP socket has the following components: source ip address, destination ip address, and IP protocol (0 - 255). UDP and TCP sockets also have a source port and a destination port. Ports range from 0 - 65535.

Access Control Entry (ACE) : One line in an access-list. Each line is a rule which specifies which packets match (based on the socket components), and what should be done with packets that match (permit or deny).

Access Control List (ACL) : A list of rules (ACEs) which, in aggregate, determine whether packets are permitted or denied. Packets which are not explicitly permitted or denied are usually implicitly denied at the end of the ACL.

Ingress ACL : This tool considers the “ingress ACL” as the ACL which applies in the direction the socket is initiated: from the source IP address to the destination IP address.

Egress ACL : This tool considers the “egress ACL” as the ACL which applies to return traffic (from the destination IP address to the source IP address). Almost all IP sockets have network traffic which goes in both directions (even if the return traffic consists entirely of acknowledgements).

Stateless ACL : A stateless ACL is an ACL which does not “remember” the state of the connection through the device. Packets are permitted or denied based solely on the individual packet and the ACL currently applied. Most IOS ACLs are stateless.

Stateful ACL : A stateful ACL is an ACL which “remembers” the state of the connection through the device. In particular, if ingress traffic is allowed by the stateful ACL, return egress traffic will automatically be allowed. ASA ACLs are stateful.
