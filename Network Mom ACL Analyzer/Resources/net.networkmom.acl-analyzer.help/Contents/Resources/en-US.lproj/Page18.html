Cisco Service Obect-Group Configuration

A note about service object-groups and the ACL Analyzer:

As of August 2019, we do not plan to support Cisco IOS-XE and ASA service object-groups with the Network Mom ACL Analyzer. While it is possible to support permit/deny analysis, supporting the arbitrary collection of protocols and ports with the duplicate line analysis is problematic. Our perception is that service object-groups are a rarely-used feature.

If the lack of this feature poses a problem to your environment, please send feedback@networkmom.net ACL samples so we can understand your use case.

Cisco ASA

https://www.cisco.com/c/en/us/td/docs/security/asa/asa98/configuration/firewall/asa-98-firewall-config/access-objects.html


A service object can contain a single protocol specification:


object service  object_name

  • service  protocol  —The name or number (0-255) of an IP protocol. Specify ip to apply to all protocols. 

  • service {icmp | icmp6 } [ icmp-type  [ icmp_code ]] —For ICMP or ICMP version 6 messages. You can optionally specify the ICMP type by name or number (0-255) to limit the object to that message type. If you specify a type, you can optionally specify an ICMP code for that type (1-255). If you do not specify the code, then all codes are used. 

  • service {tcp | upd | sctp } [source  operatorport ] [destination  operatorport ] —For TCP, UDP or SCTP. You can optionally specify ports for the source, destination, or both. You can specify the port by name or number. The operator can be one of the following: 

    • lt—less than. 

    • gt—greater than. 

    • eq—equal to. 

    • neq—not equal to. 

    • range—an inclusive range of values. When you use this operator, specify two port numbers, for example, range 100 200.

object service web

service tcp destination eq http


Service group:

object-group service  object_name

  • service-object  protocol  —The name or number (0-255) of an IP protocol. Specify ip to apply to all protocols. 

  • service-object {icmp | icmp6 } [ icmp-type  [ icmp_code ]] —For ICMP or ICMP version 6 messages. You can optionally specify the ICMP type by name or number (0-255) to limit the object to that message type. If you specify a type, you can optionally specify an ICMP code for that type (1-255). If you do not specify the code, then all codes are used. 

  • service-object {tcp | udp | tcp-udp | sctp } [source  operatorport ] [destination  operatorport ] —For TCP, UDP, or both, or for SCTP. You can optionally specify ports for the source, destination, or both. You can specify the port by name or number. The operator can be one of the following: 

    • lt—less than. 

    • gt—greater than. 

    • eq—equal to. 

    • neq—not equal to. 

    • range—an inclusive range of values. When you use this operator, specify two port numbers, for example, range 100 200. 

  • service-object object  object_name —The name of an existing service object. 

  • group-object  object_group_name —The name of an existing service object group.
