General Limitations

Network Mom ACL Analyzer supports analyzing the following types of  security  access lists:

  • Cisco IOS (including IOS-XE) IPv4 extended access-lists

  • Cisco IOS-XR IPv4 access-lists, including nested object-group support

  • Cisco ASA IPv4 extended access-lists, including nested network object-group support

  • Cisco NX-OS IPv4 L3 access-lists, including object-group support.

Network Mom ACL Analyzer has many limitations. Here is a list of _some_ of those limitations. Also see the platform-specific limitations on the following pages.

  • Socket analysis currently only supports TCP and UDP sockets.

  • IPv6 access-lists are not supported

  • The ACL analyzer converts all input to lowercase because most Cisco configuration is case-insensitive. This can cause diagnostic output to be lowercase when original input was mixed case. In a few cases Cisco may treat input in a case-sensitive manner resulting in incorrect analysis.

  • Text names for ports and protocols ( “esp”, “isakmp” and “non500-isakmp” for example) are very OS and platform specific. Network Mom ACL Analyzer does not know your precise OS or platform and cannot verify that the text name you use will be recognized on your device.

  • IOS Object-groups (for those IOS devices/versions which support them) are not supported.

  • Discontiguous “don’t care bit” configurations are reported as errors (see related help page).

  • DNS lookup of hostnames in ACL configurations*

  • Standard access-lists (although the short source-ip address form of IOS-XR ACEs are supported)

  • NAT configurations

  • Dynamic ACLs (lock and key)

  • VACLs

  • Reflexive ACLs

  • Time-based ACLs

  • Context-based access control

  • Infrastructure Protection Access Control Lists

  • Access lists which use keywords for router IP addresses, such as “infrastructure_ips”, “bgp_peer”, “router_ip”, or “interface” (ASA).

  • Access list entries which filter based on DSCP, precedence, tos, fragments, or IGMP message-type. ICMP message-types are only partially understood.

  • Understanding which L3/L2 ACL features are supported on various types of NX-OS boxes (for example, L3 ACL’s are not supported on L2 ports on a Nexus 5010).

  • Mac-address access-lists.

  • Out-of-order or duplicated sequence numbers are reported as an error and must be placed in order before socket analysis will be accurate.


* To protect the security of your ACLs, Network Mom ACL Analyzer is not allowed to make network connections. This prevents the tool from performing DNS lookups.
